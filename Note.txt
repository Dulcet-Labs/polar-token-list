# **Polar DEX Token List Registry Plan**

## **1️⃣ Token List Registry Design**

**Goal:** Have a universal token list that is **dynamic, verifiable, and plug-and-play**.

**JSON Structure:**

```json
[
  {
    "name": "PolarToken",
    "symbol": "POL",
    "decimals": 6,
    "objectId": "0x1234abcd...",
    "logoURI": "https://cdn.polar.io/pol.png",
    "verified": true,
    "addedAt": "2025-08-24T08:00:00Z",
    "verifiedBy": "adminID"
  }
]
```

* `verified: true` → ice ❄️ badge is available for Polar DEX.
* `verified: false` → token is unverified (auto-added).
* `addedAt` & `verifiedBy` → audit history.

**Hosting:**

* Serve from a **public CDN or backend API** for wallets and apps.
* Example: `https://api.polar.io/sui-token-list.json`

---

## **2️⃣ Automatic Token Addition (Unverified)**

**Goal:** Keep the token list dynamic and up-to-date.

**Steps:**

1. **Blockchain monitoring:**

   * Use a **SUI RPC node or provider** to watch for newly created token objects.
   * Pull metadata: name, symbol, decimals, object ID.

2. **Add to JSON automatically:**

```json
{
  "name": "NewToken",
  "symbol": "NTK",
  "decimals": 6,
  "objectId": "0x5678efgh...",
  "logoURI": "",
  "verified": false,
  "addedAt": "2025-08-24T08:15:00Z"
}
```

3. **Front-end display:**

* Tokens appear in dropdowns/searches as unverified.
* Users can manually input object IDs if token is missing.

**Technical needs:**

* **SUI RPC / full node access** → to query token objects.
* Backend service / cron job → periodically fetch new tokens and update JSON.

---

## **3️⃣ Team Verification (Ice ❄️ Badge)**

**Goal:** Trusted tokens get the verification badge.

**Steps:**

1. **Verification form:**

   * Token Object ID, logo, website, socials, optional audit report.

2. **Manual review:**

* Confirm token exists and metadata matches.
* Check project legitimacy (website, socials, GitHub).
* Optional: liquidity verification or audit check.

3. **Update token list JSON:**

```json
{
  "verified": true,
  "verifiedBy": "adminID",
  "addedAt": "2025-08-24T08:20:00Z"
}
```

4. **Front-end UI:**

* Show ice ❄️ badge on Polar DEX.
* Wallets/apps can optionally show it.

**Technical needs:**

* Backend admin panel → review submissions and update JSON.
* Logging → track who verified and when.

---

## **4️⃣ Front-End Integration**

* **DEX UI:**

  * Dropdowns, swap pages, token info pages.
  * Always show ice ❄️ badge for verified tokens.

* **Wallet / Third-party apps:**

  * Fetch JSON from public URL.
  * Optional badge display.

**Technical needs:**

* Front-end JS fetch → pull JSON dynamically.
* Optional caching for performance.

---
